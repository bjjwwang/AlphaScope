Feature Set

Alpha158: Contains 158 classic factors including moving averages, volatility, momentum, and correlations. Recommended for tree models (LightGBM, etc.).
Alpha158 Top 20: The 20 most return-correlated factors selected from Alpha158. Ideal for sequence models like RNN/GRU (reduces dimensionality for efficiency).
Alpha360: Uses raw OHLCV data from the past 60 days (6x60=360 dimensions). Suitable for deep learning models that auto-extract features.
Lookback window in days for time-series models. Short-term (5-10), Medium (20-30), Long-term (40-60)
4

Data Source

Daily data is most abundant and stable. Intraday data is only available in stock_data.db (~4 months).
5

Target Stock

The training pool determines which stocks the model learns from. For large-cap stocks, use "S&P 500". For meme/small-cap stocks, use "Volatile / meme stocks" to help the model learn high-volatility patterns.
6

Time Range

~8 months

Training set: The model learns patterns from this data. Recommended: 3-12 months.
Validation set: Used for early stopping (prevents overfitting). Recommended: 1-3 months.
Test set: Data the model has never seen, used to evaluate real performance.
7

Trading Strategy

Selects from top K predicted stocks daily, rotating N. Suitable for multi-stock portfolios.
When set to 0 (auto mode): the target stock is compared cross-sectionally against all stocks daily and bought when it ranks in the top 50%. This avoids guessing score magnitudes. When set manually, the raw prediction score is compared directly (typical magnitude: +/-0.01~0.03).
8

Prediction Label

Expression: Ref($close, -2)/Ref($close, -1) - 1
The prediction label determines what the model learns to predict. 1d is for short-term high-frequency trading; 5d-10d for swing trading; 20d for monthly rebalancing. Longer labels make the model focus more on trends rather than noise.

      Metrics Guide: 
      Model Return = Cumulative return from model's buy/sell signals during the test period  
      Buy & Hold = Actual price change during the same period (benchmark)  
      Excess Return = Model Return − Buy & Hold. Positive = outperformed  
      Win Rate = Profitable trades / Total trades  
      Trades = Total buy/sell signals during test period
      
Ranking: Sorted by model return (descending), then by win rate. #1 is the best model.
